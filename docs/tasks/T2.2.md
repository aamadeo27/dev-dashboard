# T2.2 — ProjectScanner

**Status:** done

## What was done

Implemented a `scanner::detect` function that inspects marker files in a project directory to determine its language and package manager. Wired the scanner into the project registration flow (`add_project`) and added a `scan_project` Tauri command that re-runs detection on demand.

## How the affected component works

### Detection logic (`scanner::detect`)

`detect(path: &Path) -> (Option<String>, Option<String>)` checks for marker files in a fixed priority order using synchronous `Path::exists()`:

1. `Cargo.toml` → `("rust", "cargo")`
2. `package.json` + `pnpm-lock.yaml` → `("typescript", "pnpm")`
3. `package.json` (alone) → `("typescript", "npm")`
4. `pyproject.toml` + `uv.lock` → `("python", "uv")`
5. `pyproject.toml` + `poetry.lock` → `("python", "poetry")`
6. `go.mod` → `("go", "gomod")`
7. none → `(None, None)`

First match wins. Synchronous I/O is acceptable because `detect` is called from `add_project` (before any async lock is released) and from `scan_project` (inside the Tokio mutex, on the thread pool implicitly).

### Registration flow

`ProjectRegistry::add_project` now calls `scanner::detect(&canonical)` after canonicalization and before constructing the `Project` struct. The returned `(language, package_manager)` pair is stored directly on the new project.

### Re-scan command

`ProjectRegistry::scan_project(id)` finds the project by id, re-runs `scanner::detect` on `project.path`, updates `language`, `package_manager`, and `last_modified`, then persists via `save()`. The Tauri command `scan_project` is a thin wrapper delegating to this registry method, following the existing T2.1 pattern.

## Decisions made within Task scope

- **`scan_project` logic lives on `ProjectRegistry`** rather than inline in the command, matching the T2.1 pattern where commands are thin wrappers over registry methods.
- **`pyproject.toml` alone (no lock file) returns `(None, None)`** — a `pyproject.toml` without `uv.lock` or `poetry.lock` is ambiguous, so no match is safer than a wrong guess.
- **`scan_project` updates `last_modified`** — the field communicates when the project record last changed; a re-scan that changes the stored values is a mutation, so updating `last_modified` is consistent with other mutating operations.
