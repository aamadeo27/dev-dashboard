# Wave 1 Review Fixes (T0.2 + T0.5)

## What was done

Applied eight review findings from the Wave 1 code review (T0.2 IPC foundation + T0.5 design tokens / global CSS). Fixes range from a Rust variant rename to frontend test hygiene.

## How the affected components work

### error.rs — AppError

`AppError` is the single application error type returned by all Tauri commands via `AppResult<T> = Result<T, AppError>`. It serializes to `{ "code": "...", "message": "...", "details": null }` via a manual `serde::Serialize` impl. The `code()` method drives the serialized `"code"` field.

### ipc/commands.rs — Tauri commands

Each `#[tauri::command]` function in this file is registered in `lib.rs` via `tauri::generate_handler![]`. The frontend calls them through `invoke("command_name", args)`. `ping_error` was added to give the IPC bridge an observable path through `AppError` serialization (T0.2 acceptance criterion).

### globals.css — monospace font-size

The `code, pre, kbd` rule uses the design token `--font-size-code: 12px` (UI spec §1.2 "Code / paths / diffs / run event text"). Previously it incorrectly used `--font-size-xs: 11px` (metadata/timestamps).

### tokens.css — warning alias

`--warning` shares the same amber value as `--accent`. The comment makes the coupling explicit so future palette changes are not missed.

### App.test.tsx — hash routing tests

Tests set `window.location.hash` before rendering to simulate HashRouter navigation. Without `afterEach` cleanup the hash leaks between tests and can cause ordering-dependent failures. The trivially-passing smoke test (`expect(document.body).toBeTruthy()`) was removed as a duplicate of the "renders S-02 Dashboard on / route" test.

### tokens.test.ts — hasToken regex

The previous `includes("--token-name:")` would match `--token-name-extra:` as a false positive. The replacement regex `(^|\s)--token-name\s*:` requires the token to be preceded by whitespace or start-of-line, preventing partial-name collisions.

## Decisions made within task scope

- The duplicate "renders without crashing on root route" test was deleted (rather than replacing its assertion) because an identical meaningful assertion already exists in "renders S-02 Dashboard on / route". Keeping two identical tests adds no coverage.
- The trailing commas inside JSX `render(...)` calls in App.test.tsx were pre-existing Biome formatting violations that became visible once the file was touched; they were corrected to keep `lint:ci` green.
- `tokens.test.ts` Node protocol imports (`"fs"` → `"node:fs"`) were pre-existing Biome violations in the same file as Fix 8; corrected together.

```
cd src-tauri

```
