# T4.5 — OrphanReaper

**Epic**: 4 — Run Execution Core  
**Type**: Backend (Rust)  
**Branch**: `feat/T4.5-orphan-reaper`  
**Worktree**: `../dev-dashboard-T4.5`

## Description

On app startup, scan all registered projects' `.claude/runs/*/meta.json`. For any with `status in (pending, running)`, check if the PID is alive AND its exe path matches the configured claude CLI. If yes, kill it. Mark all such runs `failed` with `note="Terminated (app restarted)"`.

## Acceptance Criteria

- Smoke test: launch a run, force-quit the app, relaunch → run is marked failed and the (mock) child is gone.
- Conservative: a PID alive but with a different exe is NOT killed — only kill if PID is alive AND exe path matches the configured CLI path.
- Runs with `status in (completed, failed, stopped)` are never touched.

## Key Files

- `src-tauri/src/runs/` — RunManager, RunSession, meta.json structs
- `src-tauri/src/settings.rs` — configured claude CLI path
- `src-tauri/src/lib.rs` (or `main.rs`) — startup sequence (OrphanReaper::run() must be called before RunManager handles new launches)
- `docs/kb/contracts/run.md` — meta.json contract (status enum, pid field)
- `docs/kb/rust-crates.md` — `sysinfo` crate for PID/exe matching

## Notes

- Use `sysinfo` crate to check PID liveness and exe name.
- Conservative: only kill if `exe_path == configured_claude_cli_path` (not just process name).
- Log each killed PID at `info` level: `orphan_reaper: killed PID={} run_id={} reason="app restarted"`.
