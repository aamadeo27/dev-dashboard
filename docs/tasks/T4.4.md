# T4.4 — Attached-md Context Handling

**Epic**: 4 — Run Execution Core
**Type**: Backend (Rust)
**Status**: Implemented (iteration 1) → review fixes applied

## What Was Done

When `LaunchInput.attached_md_path` is set, the file is read before any child process is spawned. If the file does not exist or cannot be read, `AppError::NotFound` is returned immediately (no run directory created, no child spawned). If the file is found, its contents are written to the child's stdin as the first write before the session handle is handed off to the background I/O loop. `attached_md_path` is already a field on the `Run` struct, so `meta.json` records it automatically via `TranscriptWriter::create`.

T1.2 probed only `claude --version`; no CLI argument for context files was found. The stdin-prepend strategy is used.

## How the Affected Component Works

### Data flow

```
Frontend invoke("launch_run", { project_id, sequence_name, attached_md_path })
  → launch_run(input, state, app_handle)
      → 1.  resolve project path
      → 1b. validate sequence_name
      → 1c. if attached_md_path is Some:
              read_attached_md(path) → Vec<u8>  (NotFound if missing, InvalidInput if > 1 MiB)
      → 2.  generate run_id
      → 3.  build + verify run_dir
      → 4.  resolve CLI path
      → 5.  build initial Run { ...attached_md_path... }
      → 6.  TranscriptWriter::create (writes meta.json with attached_md_path)
      → 7.  spawn child process (stdin = piped)
      → 7b. if attached_md_content is Some:
              stdin.write_all(&content)    ← first write to child stdin
              stdin.write_all(b"\n")       ← separator newline
      → 8.  build SessionHandle { stdin: Arc<Mutex<Option<ChildStdin>>> }
      → 9.  insert handle, spawn run_io_loop
      → return Run (status = Pending)
```

### `read_attached_md`

Pure async helper (`pub async fn`). Takes a `&Path`, calls `tokio::fs::metadata` first to detect missing files and enforce the 1 MiB cap, then `tokio::fs::read` to load the bytes. Returns:
- `Ok(Vec<u8>)` — file exists, is within limit.
- `Err(AppError::NotFound)` — file not found or not accessible.
- `Err(AppError::InvalidInput)` — file exceeds 1 MiB.

The 1 MiB cap is a safety bound: an unbounded file could exhaust memory or cause the child process to stall on a massive pipe write.

### meta.json recording

`Run.attached_md_path: Option<PathBuf>` is already in the struct definition (`runs/mod.rs`) and is serialized by `serde`. `TranscriptWriter::create` writes the initial `meta.json` from the `Run` value at step 6 — before stdin write — so the path is always recorded regardless of whether the stdin write succeeds.

### stdin write error handling

The stdin write is best-effort: a failure to write (e.g., child exits immediately) is logged at `warn` level and the run continues. This matches the existing pattern for stdin writes in `send_input`. The context not being received is unfortunate but not a reason to abort the run.

## Decisions Made Within Task Scope

- **1 MiB file size cap**: No cap was specified in the task. An unbounded read is a risk (memory exhaustion, pipe stall). 1 MiB is generous for a context file and matches typical markdown file expectations. Coder decision.
- **Best-effort stdin write**: Consistent with the existing `send_input` pattern; failure is logged but does not abort the run.
- **Read before spawn, not after**: Ensures the caller sees a clean `NotFound` error with zero side effects (no run_dir, no child process, no orphaned session). The `read_attached_md` call is at step 1c, before any filesystem writes.
- **Trailing newline after content**: Separates the context block from whatever the CLI reads next. Consistent with shell convention.

## Test scenarios

### IT-T4.4-01 — Happy path: valid attached_md_path, contents written to stdin first
- **Level**: medium (integration)
- **Preconditions**: A temp file containing known UTF-8 content exists. A real echo-like binary (`cmd /c type` on Windows, `cat` on POSIX) is available to read and forward stdin.
- **Steps**:
  1. Write `b"# Context\n\nHello from context.\n"` to `<tmpdir>/context.md`.
  2. Call `read_attached_md(&path)`.
  3. Assert `Ok(content)` where `content == original bytes`.
- **Expected**: Returns `Ok(Vec<u8>)` equal to the file bytes. No error.

### IT-T4.4-02 — Missing file returns NotFound before any side effect
- **Level**: medium (integration)
- **Preconditions**: A path that does not exist on disk.
- **Steps**:
  1. Call `read_attached_md(&nonexistent_path)`.
  2. Assert the result variant.
- **Expected**: `Err(AppError::NotFound(_))`. No partial state (no run dir created, no child spawned — verified by checking no side-effect files appear in tempdir).

### IT-T4.4-03 — Oversized file (> 1 MiB) returns InvalidInput before spawn
- **Level**: medium (integration)
- **Preconditions**: A file of 1 MiB + 1 byte.
- **Steps**:
  1. Write `[0u8; 1_048_577]` to `<tmpdir>/big.md`.
  2. Call `read_attached_md(&path)`.
- **Expected**: `Err(AppError::InvalidInput(_))`. File not read into memory.

### IT-T4.4-04 — Exactly 1 MiB file is accepted (boundary)
- **Level**: medium (integration)
- **Preconditions**: A file of exactly 1_048_576 bytes.
- **Steps**:
  1. Write `[b'a'; 1_048_576]` to `<tmpdir>/exact.md`.
  2. Call `read_attached_md(&path)`.
- **Expected**: `Ok(content)` where `content.len() == 1_048_576`.

### IT-T4.4-05 — Empty file (0 bytes) is accepted
- **Level**: medium (integration)
- **Preconditions**: A zero-byte file.
- **Steps**:
  1. Write `b""` to `<tmpdir>/empty.md`.
  2. Call `read_attached_md(&path)`.
- **Expected**: `Ok(vec![])` — empty Vec, no error.

### IT-T4.4-06 — NotFound error message contains the file path
- **Level**: medium (integration)
- **Preconditions**: A path that does not exist.
- **Steps**:
  1. Call `read_attached_md(&path_with_distinctive_filename)`.
  2. Extract the inner message from `AppError::NotFound(msg)`.
- **Expected**: `msg` contains the filename string so callers can surface a useful error.

### IT-T4.4-07 — File at 1 MiB + 1 byte: InvalidInput message mentions size
- **Level**: medium (integration)
- **Preconditions**: A file of 1_048_577 bytes.
- **Steps**:
  1. Call `read_attached_md(&path)` on the oversized file.
  2. Extract the inner message from `AppError::InvalidInput(msg)`.
- **Expected**: `msg` mentions either the byte count or "too large" so the caller can surface an actionable error.

### IT-T4.4-08 — No attached_md_path: helper is not called (boundary / null case)
- **Level**: medium (integration)
- **Preconditions**: None.
- **Steps**:
  1. Confirm `read_attached_md` is only called when `attached_md_path` is `Some`. Test the branch by calling `read_attached_md` on a legitimate path and confirming `Ok(_)` — no implicit call path for `None`.
- **Expected**: When called with a valid file, `Ok`. The `None` branch of `launch_run` never touches `read_attached_md`.

### IT-T4.4-09 — meta.json records attached_md_path via TranscriptWriter::create
- **Level**: medium (integration)
- **Preconditions**: A tempdir acting as a run directory.
- **Steps**:
  1. Construct a `Run` with `attached_md_path = Some(<path>)`.
  2. Call `TranscriptWriter::create(run_id, &run_dir, &run)`.
  3. Read `run_dir/meta.json` and parse as `serde_json::Value`.
- **Expected**: `meta["attached_md_path"]` is non-null and contains the path string.

### IT-T4.4-10 — meta.json records null when no attached_md_path
- **Level**: medium (integration)
- **Preconditions**: A tempdir acting as a run directory.
- **Steps**:
  1. Construct a `Run` with `attached_md_path = None`.
  2. Call `TranscriptWriter::create` and read `meta.json`.
- **Expected**: `meta["attached_md_path"]` is JSON `null`.

### Regression risks
- The 1 MiB check uses `meta.len()`, so a file modified between `metadata()` and `read()` could slip through on a busy filesystem. The gap is negligible in test environments but noted.
- `launch_run` reads the file at step 1c — before any filesystem writes — so a `NotFound` at that step must not create run_dir (covered by IT-T4.4-02).

## Iteration 1 Fix Pass

**Fixes Applied** (addressing review findings F-2, F-4, F-7):

1. **Cargo.toml**: Removed duplicate `chrono` and `serde_json` entries from `[dev-dependencies]`. Both are already in `[dependencies]` and Cargo deduplicates at build time, but the entries are redundant noise.

2. **commands.rs step 7b**: Added guard `if !content.is_empty()` around the stdin write block. When an empty file is read (accepted by `read_attached_md` per IT-T4.4-05), the code now skips writing an empty slice + lone newline. A spurious newline on stdin before any real prompt could confuse the Claude CLI.

3. **Test IT-T4.4-05 comment**: Updated to accurately describe the guard: "skips writing an empty content + newline to stdin" (the code now does this; prior comment claimed this happened but it did not).

4. **Test IT-T4.4-07 assertion**: Removed the overly-broad fallback check `msg.chars().any(|c| c.is_ascii_digit())`. The new assertion requires the message to contain either `"too large"` or `"bytes"`. The digit check was too permissive — it passes whenever any digit appears in the path (e.g., `/tmp/tmp1234/oversized.md`), regardless of whether the error message actually mentions size.
