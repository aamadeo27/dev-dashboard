# T4.3-sec-fixes — Security fixes for RunManager / RunSession

## What was done

Applied four security fixes to the Rust backend identified in the T4.3 review findings.
No functional behaviour was changed; only security guards and spawn ordering were adjusted.

---

## How the affected components work

### `commands.rs` — `launch_run`

`launch_run` accepts a `LaunchInput` from the frontend, resolves the project path, builds a
run directory path, spawns the Claude CLI child process, creates a session handle, and returns
an initial `Run` record. A background Tokio task (`run_io_loop`) streams stdout/stderr and
emits Tauri events.

### `commands.rs` — `send_input`

`send_input` looks up the active session by `run_id`, then writes `text + "\n"` to the child's
stdin handle.

### `session.rs` — `verify_run_dir_prefix`

Canonicalizes the project root, builds the expected prefix `<canonical>/.claude/runs/`, and
verifies lexically (after normalizing Windows backslashes) that `run_dir` starts with that
prefix. Guards against directory traversal from a malformed `run_id`.

---

## Fixes applied

### FIX-SEC-1 — Validate `sequence_name` before spawn

**File:** `src-tauri/src/ipc/commands.rs`

Added `pub fn is_valid_sequence_name(name: &str) -> bool` free function (extracted for
testability) with three rules:
- Reject empty string.
- Reject names starting with `-` (prevents `--print`, `--help`, etc. as CLI flag injection).
- Reject any character outside `[A-Za-z0-9._\-/ ]`.

Called in `launch_run` between step 1 (resolve project) and step 2 (generate run id). Returns
`AppError::InvalidInput("sequence_name: ...")` on failure.

### FIX-SEC-2 — Cap `send_input` text length

**File:** `src-tauri/src/ipc/commands.rs`

Added a 64 KiB (`65_536` byte) limit in `send_input` before writing to stdin. Returns
`AppError::InvalidInput("send_input: text too large ...")` if exceeded.

### FIX-SEC-3 — Add trailing separator to Windows path prefix check

**File:** `src-tauri/src/runs/session.rs`

Changed `prefix_norm` construction from:
```rust
let prefix_norm = expected_prefix.replace('\\', "/");
```
to:
```rust
let prefix_norm = format!("{}/", expected_prefix.replace('\\', "/"));
```

Without the trailing `/`, a path like `.claude/runs-evil/abc` would falsely pass as a prefix
of `.claude/runs` because `"runs-evil".starts_with("runs")` is true.

### FIX-SEC-4 — Spawn I/O task before inserting session into map

**File:** `src-tauri/src/ipc/commands.rs`

Previously: map insert (step 8) then `tokio::task::spawn` (step 9).
Now: `tokio::task::spawn` first (capturing `JoinHandle` with `let _ =`), then map insert.

This closes a race where `stop_run` could fire a cancellation token before the background
task's `select!` loop was active. The task is now live before any caller can observe the
handle in the map.

## Decisions made within task scope

- `is_valid_sequence_name` is `pub` (not `pub(crate)`) to keep it importable by integration
  tests in `src-tauri/tests/` without a Tauri runtime — consistent with the pattern used for
  `resolve_cli_path`.
- The `JoinHandle` from `tokio::task::spawn` in FIX-SEC-4 is discarded with `let _ =`. The
  task is fire-and-forget by design (it removes itself from the sessions map on completion);
  storing the handle would require a structural change outside this task's scope.
- The 64 KiB cap in FIX-SEC-2 is placed after the `NotFound` guard but before the stdin lock,
  so oversized inputs are rejected cheaply without acquiring the mutex.

## How to test / verify

### Unit tests (Rust)

```
cargo test --lib -p dev-dashboard-lib
```

New tests added:

**`src-tauri/src/ipc/commands.rs` — `mod tests`:**
- `is_valid_sequence_name_rejects_empty`
- `launch_run_rejects_sequence_name_with_leading_dash` (covers `-`, `--print`, `--help`, `-x`)
- `is_valid_sequence_name_rejects_unsafe_characters` (`;`, `&`, `|`, `$`, `\n`, `\x00`)
- `is_valid_sequence_name_accepts_valid_names` (`my-sequence`, `sub/dir`, `seq 1`, etc.)
- `send_input_size_guard_accepts_max_bytes` (exactly 65536 bytes)
- `send_input_size_guard_rejects_over_max_bytes` (65537 bytes)
- `send_input_size_guard_accepts_empty` (0 bytes)

**`src-tauri/src/runs/session.rs` — `mod tests`:**
- `verify_run_dir_prefix_rejects_runs_sibling_dir` (`.claude/runs-evil/abc` must return `PermissionDenied`)

### Manual smoke test

1. Launch the app, register a project.
2. Attempt `launch_run` with `sequence_name = "--help"` — must return `InvalidInput`.
3. Attempt `send_input` with a 70 KiB string — must return `InvalidInput`.
4. Normal sequences with names like `"my-sequence"` or `"sub/dir"` must launch successfully.
