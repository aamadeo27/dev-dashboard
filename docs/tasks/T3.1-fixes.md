# T3.1-fixes ‚Äî SequenceLoader Review Fixes

## What was done

Applied 9 review fixes to the T3.1 SequenceLoader implementation. Fixes span correctness (UTF-8 handling), security (symlink rejection, file size cap), resilience (warn-and-skip instead of scan-abort), and quality (Default derive, deterministic timestamps, span entry, deduplication helper).

## How the affected component works

`SequenceLoader` (`src-tauri/src/sequences/mod.rs`) is an in-memory cache keyed by `project_id`. Each cache entry stores the directory mtime and the scanned `Vec<Sequence>`. On `load_all`, the directory mtime is compared to the cached value; a mismatch or cache miss triggers `scan_sequences_dir`. `refresh` evicts the cache entry first, then calls `load_all`.

`scan_sequences_dir` iterates `<project_path>/.claude/sequences/*.md` and for each file:
1. Checks the `.md` extension
2. Calls `symlink_metadata` to get stats without following symlinks ‚Äî rejects symlinks
3. Checks file size against `MAX_SEQ_FILE_BYTES` (1 MiB)
4. Gets mtime from the same metadata object (warn-and-skip on failure)
5. Reads raw bytes and decodes as UTF-8, falling back to lossy decode on invalid bytes
6. Calls `extract_description` to get the first non-heading, non-blank line
7. Pushes `Sequence { name, description, path, mtime }`

Results are sorted alphabetically by name.

`system_time_to_utc` converts `SystemTime` to `DateTime<Utc>`, returning epoch on overflow (deterministic) instead of `Utc::now` (non-deterministic ‚Äî caused phantom cache invalidation).

The `list_sequences` and `refresh_sequences` Tauri commands in `commands.rs` now use `get_sequence_project_path` ‚Äî a `pub(crate)` helper ‚Äî for the shared project-path lookup.

## Decisions made within Task scope

- Used `lmeta` (from `symlink_metadata`) for both the symlink check, size check, and mtime ‚Äî eliminating the separate `tokio::fs::metadata` call that existed before. This is safe because symlinks are rejected before the size/mtime reads, so `lmeta` always describes the actual file.
- `get_sequence_project_path` is `pub(crate)` (not `pub`) ‚Äî sufficient for internal command use and consistent with the codebase's visibility conventions.
