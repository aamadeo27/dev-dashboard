# T4.2 — TranscriptWriter

## What was done

Implemented `TranscriptWriter` in `src-tauri/src/runs/transcript.rs`.  
It is the per-run file I/O primitive that durably persists run events, raw process output, and run metadata under a dedicated `run_dir`.

## How the component works

### Files managed

All files live under a single `run_dir` (`<project>/.claude/runs/<run-id>/`):

| File | Format | Write strategy |
|---|---|---|
| `meta.json` | Pretty-printed JSON of `Run` | Atomic: write to `meta.json.tmp` then `rename` |
| `transcript.jsonl` | One compact JSON `RunEvent` per line | Append + flush per event |
| `raw.log` | Raw bytes (stdout/stderr) | Append, no per-write flush |

### Data flow

1. **`create(run_dir, initial_meta)`** — creates the directory tree with `create_dir_all`, writes `meta.json` atomically, opens `transcript.jsonl` and `raw.log` with `create(true).append(true)`.
2. **`append_event(event)`** — serializes with `serde_json::to_string` (compact, no embedded newlines), appends `\n`, then flushes. Each call is a sync point: on a kill, at most one in-flight event is lost; all prior events are durable.
3. **`append_raw(bytes)`** — calls `write_all` on `raw.log`. No flush; OS buffering is acceptable here.
4. **`update_meta(run)`** — pretty-prints `Run`, writes to `meta.json.tmp`, renames over `meta.json`. Atomic on POSIX; best-effort on Windows (rename is not truly atomic on Windows but is the standard approach here).
5. **`close(self)`** — flushes both `transcript` and `raw` handles, then drops them.

### Contracts

- `AppResult<T>` propagates I/O errors as `AppError::Io` (via `From<std::io::Error>`).
- Serialization failures become `AppError::Internal`.
- Concurrent writers for different `run_dir`s are safe because each instance owns its own file handles and writes only to its own directory.

## Decisions made within Task scope

- `close()` flushes `raw.log` even though `append_raw` doesn't flush per-write, to drain any OS write buffer before the file handle is dropped.
- The `run_dir` field is stored on the struct (as `PathBuf`) so `update_meta` can construct the tmp path without needing it passed in on every call.
- Serde JSON serialization errors use `AppError::Internal` rather than `AppError::Parse` because they represent a programmer error (all types derive `Serialize`/`Deserialize`), not a user-facing parse failure.

### Contracts updated by this pass

- `TranscriptWriter::create()` signature is now `create(run_id: &str, run_dir: &Path, initial_meta: &Run)`. Callers (T4.3) must add the `run_id` argument.
- Serialization failures are now `AppError::Parse` (was `AppError::Internal`). Error-handling code that pattern-matches on the variant will need to be updated if it expected `Internal`.
- `update_meta` now takes `&self` instead of `&mut self`; no borrow-checker impact for existing callers since `&self` is less restrictive.
