# T4.6 — RetentionPruner

**Epic**: 4 — Run Execution Core  
**Type**: Backend (Rust)  
**Branch**: `feat/T4.6-retention-pruner`  
**Worktree**: `../dev-dashboard-T4.6`

## Description

At startup and on a 24h timer, walk each project's runs dir. Apply both retention rules from settings (age days; total-size MB per project). Delete oldest first to satisfy both. Skip runs in any non-terminal state. Emit `info` log lines per deletion.

## Acceptance Criteria

- With 600 MB of fake runs and a 500 MB cap, oldest are pruned to bring it under.
- With 31-day-old runs and a 30-day cap, those are pruned.
- Active runs (status pending or running) are never deleted.
- Pruner runs at startup + once every 24h (tokio timer).

## Key Files

- `src-tauri/src/runs/` — run storage layout, meta.json format
- `src-tauri/src/settings.rs` — `retention_max_days`, `retention_max_mb` settings fields
- `src-tauri/src/lib.rs` (or `main.rs`) — startup sequence (RetentionPruner::run() at startup + schedule 24h task)
- `docs/kb/contracts/run.md` — meta.json contract (status, created_at, size hints)
- `docs/kb/contracts/settings.md` — settings contract (retention fields)

## Notes

- Sort candidate runs by `created_at` ascending (oldest first) before pruning.
- Per-deletion log: `retention_pruner: pruned run_id={} project={} reason={age|size} size_mb={} age_days={}`.
- Total-size calculation: sum all run dirs under a project (including transcript.jsonl + raw.log).
