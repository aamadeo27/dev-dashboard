## How the component works

### Data flow

Props come in from a parent (Dashboard, T2.6). The component derives display values from them synchronously:

- `gitEdgeColor(gitStatus)` — maps GitStatus fields to a CSS variable string for the card's 4px left border.
- `gitBadgeText(gitStatus)` — maps GitStatus fields to a human-readable badge string.
- `quickRunTitle` — derived inline from `lastRun === null` check.
- `OutcomeBadge` — internal helper component mapping `RunStatus` to colored span.
- `formatRelative(isoString)` — converts an ISO timestamp to a relative string (e.g. "5m ago").

### Five states

1. **Normal** — shows header (name + quick-run), optional tag chips, git row (branch + badge), meta row (last run + path).
2. **Missing** — `project.is_missing === true` renders a red banner with `AlertTriangle` icon; git/meta rows are hidden.
3. **Loading skeleton** — `ProjectCardSkeleton` export; renders animated placeholder boxes (no live data, `aria-busy="true"`).
4. **Running** — `activeRun` prop set; a pulsing "Running" badge appears in the header row.
5. **No prior runs** — `lastRun === null`; meta row shows "Never", quick-run button title is "Pick a sequence".

### Pulse animation

Two CSS keyframes (`pc-pulse` for the running badge, `pc-skeleton` for skeleton shimmer) are injected into `document.head` once on module load via `ensurePulseKeyframes()`. The injection is guarded by an id check so it runs at most once per document.

### Event handling

- Card `div` receives `onClick` → `onCardClick` and `onContextMenu` → `onContextMenu`.
- Quick-run button calls `e.stopPropagation()` before calling `onQuickRun`, so card click does not also fire.
- Running badge button calls `e.stopPropagation()` to prevent spurious card clicks.

## Decisions made within Task scope

- `formatRelative` is a module-level helper (not exported). It computes relative timestamps at render time from `Date.now()`. This is appropriate for a presentational component — the parent controls re-render frequency.
- `OutcomeBadge` is a file-private component. It is not exported; the task only required `ProjectCard` and `ProjectCardSkeleton` as named exports.
- The `skeletonCard` style omits `border` shorthand (which conflicts with `borderLeft`) — uses `border-radius` and `border-left` directly to avoid the style merge warning seen in Settings tests.
- The `border` shorthand on the main card sets the 1px border, and `borderLeftColor` is overridden inline to avoid the shorthand/longhand conflict.
