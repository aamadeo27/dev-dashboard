# T3.1 — SequenceLoader

## What was done

Implemented `SequenceLoader` — an in-memory, mtime-invalidated cache for
`.claude/sequences/*.md` files within a project tree. Wired it into `AppState`,
added two Tauri commands (`list_sequences`, `refresh_sequences`), added two
TypeScript wrappers, and wrote unit tests at both the Rust and TypeScript layers.

## How the affected component works

### Data model

`Sequence` (pre-existing stub in `sequences/mod.rs`) is the IPC and in-memory
type:

```rust
pub struct Sequence {
    pub name: String,        // filename minus ".md"
    pub description: String, // first non-heading paragraph; "(No description)" fallback
    pub path: PathBuf,       // absolute path to the .md file
    pub mtime: DateTime<Utc>,// file mtime (for future use by callers)
}
```

### SequenceLoader

Holds an in-memory `HashMap<String, (DateTime<Utc>, Vec<Sequence>)>` keyed by
`project_id`. The `DateTime<Utc>` is the sequences-directory mtime at scan time.

**`load_all(project_id, project_path)`**:
1. Resolves `<project_path>/.claude/sequences/`.
2. If the directory does not exist → returns empty `Vec`, no error.
3. Reads the directory mtime via `tokio::fs::metadata`.
4. Cache hit (mtime unchanged) → returns the cached `Vec<Sequence>`.
5. Cache miss or mtime changed → calls `scan_sequences_dir`, updates cache.

**`refresh(project_id, project_path)`**:
1. Removes the cache entry for `project_id`.
2. Delegates to `load_all`, which forces a full re-scan.

**`scan_sequences_dir`** (private async fn):
- Reads all directory entries, filters to `.md` files.
- For each file: reads mtime, reads UTF-8 content, calls `extract_description`.
- Non-`.md` files are silently ignored. Unreadable files are logged and skipped.
- Result is sorted alphabetically by name.

**`extract_description`** (pure fn, `pub(crate)`):
- Normalises Windows line endings (`\r\n` → `\n`, bare `\r` → `\n`).
- Iterates lines; skips blank lines and lines starting with `#` (headings).
- Returns the first remaining line, trimmed.
- Falls back to `"(No description)"` if no qualifying line is found.

### AppState integration

`sequence_loader: Arc<Mutex<SequenceLoader>>` added to `AppState`.
`SequenceLoader::new_arc()` is called once during `run()` before `AppState` is
constructed. Command handlers lock the mutex, call the loader, and return.

### IPC commands (Tauri)

Two thin `#[tauri::command]` shims in `ipc/commands.rs`:

```
list_sequences(project_id: String)    -> Vec<Sequence>
refresh_sequences(project_id: String) -> Vec<Sequence>
```

Both look up the project path from `state.projects` (locked separately to avoid
holding the projects lock during I/O), then lock `state.sequence_loader`.
Both return `AppError::NotFound` if the project_id is unrecognised.

### TypeScript wrappers

Two wrappers in `src/ipc/commands.ts`:

```ts
listSequences(projectId: string): Promise<Sequence[]>
refreshSequences(projectId: string): Promise<Sequence[]>
```

`Sequence` is imported from `./bindings` (already pre-populated in bindings.ts).

## Decisions made within Task scope

- `extract_description` is `pub(crate)` rather than private so the unit tests
  in the same file can call it without a `pub` leakage to external crates.
- `system_time_to_utc` uses `DateTime::from_timestamp` with a subsecond nanos
  component so that two files written in the same second but different nanoseconds
  produce distinct mtimes (important for cache correctness on high-resolution
  filesystems). Falls back to `Utc::now()` on overflow (unreachable in practice).
- The projects lock is acquired and released before acquiring the sequence_loader
  lock. This prevents a deadlock if any future code path holds sequence_loader
  and tries to acquire projects.
- `SequenceLoader::new_arc()` is a convenience constructor that returns
  `Arc<Mutex<Self>>` directly, matching the `AppState` field type and avoiding
  boilerplate at the call site in `lib.rs`.
- Files that fail `tokio::fs::metadata` or `read_to_string` are logged at
  `WARN` level and skipped rather than propagating an error. A partially-readable
  directory should not prevent the caller from seeing the sequences that are
  readable.
