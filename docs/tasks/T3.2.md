# T3.2 — useSequences hook + SequenceRow component

## What was done

Implemented the `useSequences` TanStack Query hook, the `SequenceRow` presentational
component, and the `SequenceList` container component that wires them together.
Added 21 unit tests across two test files. Replaced the empty stub in
`src/hooks/useSequences.ts`.

## How the affected components work

### `useSequences` hook

Wraps `listSequences(projectId)` via `useQuery`. The query is disabled when
`projectId` is falsy (empty string). The query key is
`["sequences", projectId]` — keyed per project so different projects get
independent cache entries. Returns the standard TanStack Query shape
(`data`, `isLoading`, `error`, `refetch`).

### `SequenceRow`

Pure presentational component. Receives a `Sequence`, an `onRun` callback,
an optional `onSelect` callback, and an optional `selected` boolean.

- Container is `role="listitem"`, keyboard-navigable (`tabIndex=0` when
  `onSelect` is provided), and handles Enter/Space via `onKeyDown`.
- Hover state tracked with local `useState` + `onMouseEnter`/`onMouseLeave`
  for inline style switching (`--bg-hover`).
- Selected state: `--primary-dim` background + `--primary` border.
  Both `data-selected` attribute and `aria-selected` attribute are set so
  tests and assistive technology can observe the state.
- Description uses `wordBreak: 'break-word'` — no `overflow: hidden` or
  `textOverflow: ellipsis` — so long descriptions wrap cleanly.
- [Run] button calls `e.stopPropagation()` before `onRun(sequence)` so
  row click (select) does not fire when the button is clicked.

### `SequenceList`

Container component; `props: { projectId: string }`.

Data flow:
1. Calls `useSequences(projectId)` from the hook.
2. Renders loading / error / empty / populated states.
3. Tracks `selectedId: string | null` in local `useState`; passed as
   `selected={selectedId === seq.name}` to each `SequenceRow`.
4. `onRun` stub: `console.log('Run sequence:', seq.name)` — RunManager
   is Epic 4 and is intentionally not implemented here.

Empty state text is exactly `"No sequences found."` per UI spec §5.3/§5.6.

## Decisions made within Task scope

- `SequenceRow` uses local `useState` for hover rather than CSS `:hover`
  pseudo-classes, keeping style logic entirely in inline styles consistent
  with the `ProjectCard.tsx` pattern.
- `aria-selected` is set on the `role="listitem"` element (not `role="option"`,
  which requires a `role="listbox"` parent). This keeps the component usable
  in both S-03 and S-06 contexts without forcing a `listbox` container.
- `data-selected` attribute is set alongside `aria-selected` so tests can
  query it with `getAttribute("data-selected")` without relying on
  `aria-selected` string representation quirks across browsers.
- `tabIndex` is only set when `onSelect` is provided. When `SequenceRow` is
  used in a non-interactive context it remains unfocusable via tab.
- The `SequenceList` uses `seq.name` as the React key and as the `selectedId`
  discriminator. The `Sequence` type has no `id` field; `name` is the natural
  unique key (sequence names are unique within a project, matching the
  filesystem constraint).

## Acceptance criteria met

- Empty state text is `"No sequences found."` (§5.3/§5.6).
- Long descriptions wrap cleanly: `wordBreak: 'break-word'`, no
  `overflow: hidden`, no `textOverflow: ellipsis`.
- Selected row uses `--primary-dim` background + `--primary` border (§5.6).
- Hover uses `--bg-hover` (§1.1 / §6 component table).
- No direct `invoke()` in components — hooks only (KB §6.1).
- No Tailwind — CSS vars from `tokens.css` only.
