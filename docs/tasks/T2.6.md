## What was done

## How the affected component works

### Data flow

1. On mount, `verifyClaudeCli(undefined)` fires. If `found` is false, the user is redirected to `/setup`.
2. `useProjects()` fetches the project list via TanStack Query (`listProjects` IPC command).
3. `useSettings()` fetches persisted settings (view mode, etc.) via TanStack Query.
4. `useGitStatusListener()` subscribes to the `git:updated` Tauri event once for the Dashboard lifetime; each event updates the Zustand store in `useGitStatus`.
5. `filteredProjects` is derived from `projects`, `search`, and `selectedTags` via `useMemo`.
6. `useVisibleProjects(filteredProjects.map(p => p.id))` debounces a call to `setVisibleProjects` on the backend, so the git poller only polls what is visible.

### Per-card git status

`ProjectCardWrapper` is a file-private component that calls `useGitStatus(project.id)` (reads from the Zustand store) and passes the result down to the pure `ProjectCard`. This pattern is required because hooks must be called unconditionally at the component level, and we need one hook call per project id.

### Content states

- **Loading**: 6 `ProjectCardSkeleton` placeholders in the grid/list layout.
- **Empty (no projects)**: centered "No projects yet." message and "Add your first project" button.
- **No results (filtered empty)**: centered "No projects match your search."
- **Normal**: `ProjectCardWrapper` instances in the active view layout.

### View mode persistence

`handleToggleView` calls `updateSettings` with a `SettingsPatch` where `view_mode` is set and all other fields are `null` (meaning "no change" in the backend's partial-update semantics). `viewMode` is read from `settings?.view_mode` with a `"Grid"` fallback.

### Add project

`handleAddProject` opens a native directory picker via `@tauri-apps/plugin-dialog`. On selection it calls `addProject(path)` then invalidates the `["projects"]` query so the list refreshes. Backend errors surface via `toast:show` events (not handled in the UI).

## Decisions made within Task scope

- `gitStatus` from `useGitStatus` returns `GitStatus | null`. `ProjectCard` accepts `GitStatus | undefined`. The wrapper converts `null` to `undefined` via `gitStatus ?? undefined`.
- The `rerender` call in the last test wraps in a fresh `QueryClient` to avoid query caching side effects across re-renders; this does not affect the `verifyClaudeCli` once-on-mount assertion.
- `biome-ignore lint/suspicious/noArrayIndexKey` suppressor added to skeleton array render since skeleton items have no meaningful identity.

Manual smoke test in Tauri dev mode:
1. `pnpm tauri dev`
2. Dashboard loads; top bar shows "dev-dashboard" in violet, settings gear on the right.
3. With no projects: empty state shows "No projects yet." and "Add your first project" button.
4. Click "Add Project" or "Add your first project" — native folder picker opens.
5. Add a project; card appears in Grid mode.
6. Toggle to List — cards stack vertically.
7. Type in search box — cards filter by name/path.
8. Click a tag chip — filters to projects with that tag.
9. Remove claude CLI or set an invalid path — Dashboard redirects to `/setup`.
