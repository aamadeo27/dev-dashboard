## How the affected component works

### Frontend routing (`src/App.tsx`)

`BrowserRouter` wraps `Routes`. Each `Route` maps a path to a placeholder `div`. The root `/` redirects to `/projects`. Route paths follow the screen IDs from the UI spec (S-01 through S-07). Real screen components will replace the placeholder divs in later tasks.

### Design tokens (`src/styles/tokens.css`)

All CSS custom properties are on `:root`. Every token is named after its semantic role. `global.css` imports `tokens.css` then applies resets/base styles using those variables. The `@media (prefers-reduced-motion: reduce)` block zeroes all animation/transition durations app-wide.

### Rust module stubs (`src-tauri/src/`)

`lib.rs` declares the module tree with `pub mod`. Each domain directory has a `mod.rs` stub. The `ipc/` directory has `mod.rs` re-exporting `commands` and `events`. No business logic exists yet — stubs are comment-only. The `greet` command from the original scaffold has been removed; the `invoke_handler![]` macro now takes an empty list.

## Decisions made within task scope

- **Removed `greet` command**: The default scaffold's `greet` command was deleted from `lib.rs` because it has no role in the project. `invoke_handler![]` is now empty. This keeps clippy clean (no unused code warnings).
- **Added `src/utils/placeholder.test.ts`**: Vitest exits with code 1 when no test files are found. The task acceptance criterion says "vitest runs (0 tests is fine, must not error)." A single trivial test satisfies this without violating any upstream spec. This file should be deleted once real tests exist.
- **Added `"types": ["vitest/globals"]` to tsconfig.json**: Required for `describe`/`it`/`expect` to type-check without importing them. Vitest's `globals: true` config option enables the runtime globals; tsconfig needs the type declaration to accompany it.
- **Biome auto-fixed CRLF**: The scaffold template generated files with Windows CRLF line endings. Running `pnpm lint` (with `--write`) converted them to LF, matching Biome's formatter expectations.
