# T4.3 — RunManager + RunSession

## What was done

Implemented the `RunManager` and `RunSession` components that manage active Claude CLI child processes. Added three Tauri commands (`launch_run`, `stop_run`, `send_input`) that let the frontend start, stop, and interact with Claude CLI runs. A background Tokio task per run streams stdout through the `EventParser`, writes to the `TranscriptWriter`, and emits Tauri events to the frontend.

## How the affected component works

### Data flow

```
Frontend                  Rust Core                        Child (claude CLI)
   |                          |                                    |
   |-- launch_run(input) ---->|                                    |
   |                          |-- spawn claude --sequence-name --->|
   |                          |   (cwd = project_path)             |
   |                          |   write meta.json (Pending)        |
   |<-- Ok(Run{Pending}) -----|                                    |
   |                          |                                    |
   |                     [background task]                         |
   |                          |   update meta.json (Running)       |
   |<-- run:started event ----|                                    |
   |                          |<-- stdout bytes -------------------|
   |                          |   feed to EventParser              |
   |                          |   write transcript.jsonl           |
   |<-- run:event ------------|                                    |
   |                          |<-- stderr bytes -------------------|
   |                          |   write raw.log                    |
   |                          |                                    |
   |-- send_input(id, "x") -->|-- write to child stdin ----------->|
   |<-- Ok ------------------|                                    |
   |                          |                                    |
   |-- stop_run(id) -------->|-- cancel token -> child.kill() --->|
   |<-- Ok ------------------|                                    |
   |                          |<-- exit code ----------------------|
   |                          |   update meta.json (status)        |
   |                          |   flush + close transcript         |
   |<-- run:finished ---------|                                    |
```

### Key components

**`RunManager`** (`runs/manager.rs`): Holds an `Arc<Mutex<HashMap<String, Arc<SessionHandle>>>>` mapping run IDs to active session handles. The sessions map is passed by `Arc` clone into background tasks so they can self-remove on exit.

**`SessionHandle`** (`runs/manager.rs`): Per-run shared state:
- `cancel: CancellationToken` — fire to stop the run
- `stdin: Arc<Mutex<Option<ChildStdin>>>` — write to child stdin
- `run: Arc<Mutex<Run>>` — mutable run state updated by the I/O task

**`run_io_loop`** (`runs/session.rs`): The background `tokio::task`. Uses `tokio::select!` to read stdout and stderr concurrently. On stdout EOF or cancellation, drains remaining output, waits for child exit, determines final status, updates meta.json, closes the transcript writer, emits `run:finished`, and removes itself from the sessions map.

**Path security**: `validate_run_id` rejects anything not matching `[A-Za-z0-9._-]`. `verify_run_dir_prefix` canonicalizes the project path and verifies the run directory is under `<project>/.claude/runs/` using a prefix check with normalized separators.

### CLI invocation

```
<cli_path> <sequence_name>
```

No `--print` or other flags are added — the sequence name is passed as the sole positional argument. `cwd` is set to `project_path`. The child inherits no special environment beyond the system default.

### UserInput in transcript

`send_input` validates the input length and then sends the text over an `mpsc::unbounded_channel` to the background I/O loop, which writes a `UserInput` event to the transcript and emits `run:event` to the frontend. Stdin write follows the channel send. If the channel is closed (I/O loop already exited), `send_input` returns `AppError::InvalidInput("run is not accepting input")` without attempting the stdin write.

### Cancellation on Windows

`child.kill().await` is used — tokio handles the platform-specific signal (SIGKILL on Unix, TerminateProcess on Windows).

## Decisions made within task scope

1. **CLI invocation args**: `<cli_path> <sequence_name>` — single positional argument, no extra flags. This is the minimal invocation; the sequence file governs behavior.

2. **I/O loop buffer size**: 8192 bytes per read, per the task spec. This minimizes syscall overhead without excessive memory use.

3. **UserInput written to transcript via channel**: `send_input` sends the input text over an `mpsc::unbounded_channel` to the I/O loop, which holds the `TranscriptWriter`. The loop writes the `UserInput` event and emits `run:event`. The transcript writer is owned exclusively by the I/O task; the channel is the safe cross-task bridge. Order: validate length → send to channel → write to stdin.

4. **Sessions not cleaned on `stop_run`**: `stop_run` fires the cancellation token but does NOT remove the session from the map. The background I/O task removes itself after the child exits (step 8 of `run_io_loop`). This prevents a race where `send_input` runs concurrently.

5. **`verify_run_dir_prefix` uses normalized separators**: On Windows, `Path::starts_with` compares component-by-component, but the run_dir may not exist yet (created by `TranscriptWriter::create`). We normalize `\` to `/` for a string prefix check after canonicalizing only the project root.

6. **`final_status` for exit code != 0 (not cancelled)**: Mapped to `Failed`, as the task spec defines.

7. **`list_runs` and `get_run` deferred**: KB §5.3 lists `list_runs` and `get_run` as run commands. These are not implemented in T4.3. They depend on run persistence (scanning `<project>/.claude/runs/` directories and reading `meta.json` files), which is scoped to a later task. T4.3 only manages in-memory active sessions; completed runs are not queryable until the persistence layer is in place.
