# T2.5-fixes-3: ProjectCard review fixes (iteration 3)

## What was done

Applied four review findings to `src/components/ProjectCard.tsx` without changing any behavior, props interface, exports, or test expectations.

## Fixes applied

### FIX-1 (MEDIUM): `runningBadge` animation guarded by `reducedMotion`

The `styles.runningBadge` object previously hardcoded `animation: "pc-pulse 1.5s ease-in-out infinite"`. This made it inconsistent with `OUTCOME_STYLES["Running"]`, which already applied the same animation conditionally. The fix removes the hardcoded property and spreads the same conditional:

```ts
...(reducedMotion ? {} : { animation: "pc-pulse 1.5s ease-in-out infinite" }),
```

The `reducedMotion` const is evaluated once at module scope and is `false` in jsdom (test env does not implement `matchMedia`), so all existing tests are unaffected.

### FIX-2 (LOW): Removed `// FIX-N:` comment prefixes

Every inline comment of the form `// FIX-N: ...` was stripped of its task-tracking prefix. Explanatory text that had value was kept; only the `FIX-N:` label was removed. This eliminates stale task references from production source.

### FIX-3 (LOW): Inlined `handleQuickRun`

The named `handleQuickRun` function declaration inside the component body created a new function reference on every render. It was replaced with an inline arrow directly in the JSX:

```tsx
onClick={(e) => { e.stopPropagation(); onQuickRun?.(); }}
```

The named declaration was removed entirely. Behavior is identical.

### FIX-4 (LOW): Eliminated `projectNameMissing` style duplication

`projectNameMissing` previously repeated all six properties from `projectName` and added `color` and `textDecoration`. A shared `projectNameBase` const was extracted above the `styles` declaration. Both entries now spread from it:

```ts
const projectNameBase: React.CSSProperties = { /* shared properties */ };

projectName:        { ...projectNameBase, color: "var(--text-primary)" },
projectNameMissing: { ...projectNameBase, color: "var(--text-disabled)", textDecoration: "line-through" },
```

## How the affected component works

`ProjectCard` is a pure presentational component (no hooks beyond `useMemo`, no Zustand, no `invoke()`).

- `reducedMotion` is evaluated once at module load from `window.matchMedia`. It gates animation properties in both `OUTCOME_STYLES.Running` and `styles.runningBadge`.
- `OUTCOME_STYLES` is a module-level lookup table keyed by `RunStatus` — avoids per-render object allocation.
- `projectNameBase` is a module-level const holding shared CSS for the project name span; `styles.projectName` and `styles.projectNameMissing` spread it and override only what differs.
- The quick-run button uses an inline arrow; `e.stopPropagation()` prevents the card's `onCardClick` from firing.
