# T2.3 — GitPoller

**Status:** done

## What was done

Added a background git status poller to the Tauri backend. The poller periodically runs `git2`-based status checks on all projects currently visible in the frontend viewport, caches the results in memory, and pushes updates to the frontend via a `git:updated` Tauri event.

## How the affected component works

### Data flow

1. On app startup `lib.rs::run()` constructs a `GitPoller` and stores it in `AppState`.
2. `projects::git::start()` is called from `.setup()`, registering a window focus/blur handler and spawning a background `tokio::spawn` loop.
3. The poll loop waits `git_poll_interval_secs` of *unpaused* time (500 ms ticks; blur resets the counter).
4. Each cycle: snapshot `visible` IDs → for each ID look up its path from `ProjectRegistry` → call `git_status_for_path` via `spawn_blocking` (git2 is synchronous) → store result in `statuses` cache → emit `git:updated` event.
5. The frontend can call `get_git_status(id)` to read the cache immediately (returns `null` until first poll), and `set_visible_projects(ids)` to control which projects are polled.

### Key types

- `GitPoller` — holds three `Arc`-wrapped fields: `statuses` (HashMap cache), `visible` (HashSet of IDs), `paused` (AtomicBool).
- `GitStatus` — snapshot struct with `branch`, `is_clean`, `dirty_files`, `ahead`, `behind`, `last_polled`, `error`.
- `GitUpdatedPayload` — local struct emitted with each `git:updated` event; shape `{ id, status }`.

### Ahead/behind

Uses `refs/remotes/origin/<branch>` as the upstream reference. Returns `(0, 0)` if no upstream exists.

## Decisions made within Task scope

- `git_status_for_path` is `pub` (not `pub(crate)`) to allow unit tests to call it directly in the `#[cfg(test)]` block inside the same module.
- The `GitUpdatedPayload` struct is defined inline inside the `start()` function body (not at module level) to keep it local — it is only ever used in that one `emit` call.
- `include_ignored(false)` on the `StatusOptions` because ignored files should not contribute to "dirty" count.

```
cd src-tauri
cargo test projects::git::tests
```
Tests cover:
- `git_status_for_path_non_git_dir_returns_error` — temp dir with no `.git` → error Some
- `git_status_for_path_nonexistent_path_returns_error` — absolute non-existent path → error Some
- `git_poller_new_default_state` — statuses empty, visible empty, paused false
- `visible_set_replace_updates_correctly` — simulates `set_visible_projects` by direct mutex mutation
- `pause_accumulator_resets_when_paused` / `stays_zero_when_always_paused` — loop logic invariants

### Manual smoke test
1. Launch app with `pnpm dev:local`.
2. Open DevTools and listen: `window.__TAURI_INTERNALS__.invoke('get_git_status', { id: '<id>' })`.
3. Call `window.__TAURI_INTERNALS__.invoke('set_visible_projects', { ids: ['<id>'] })`.
4. Wait `git_poll_interval_secs` seconds; check that the cache populates and `git:updated` fires.
