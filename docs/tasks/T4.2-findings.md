# T4.2 — TranscriptWriter: Review Findings

## Iteration 1

### Performance Reviewer

**HIGH**

- **H1-perf** `transcript.rs:69` — Per-event `flush()` in `append_event` delegates to a `spawn_blocking` + OS syscall on every streaming event. At high event rates this caps throughput to ~200–2000 events/sec. **Fix**: Remove `flush()` from `append_event`; `close()` already flushes.
- **H2-perf** `transcript.rs:13–14, 41–53` — `tokio::fs::File` has no write buffer; every `write_all` crosses the blocking-pool boundary. **Fix**: Wrap `transcript` and `raw` handles in `tokio::io::BufWriter`.

**MEDIUM**

- **M1-perf** `transcript.rs:65–68` — `serde_json::to_string` allocates a `String`, then `push('\n')` may reallocate. Use `serde_json::to_vec` + `push(b'\n')` to avoid the extra step.
- **M2-perf** `transcript.rs:82–83` — `update_meta` re-allocates `PathBuf` for `tmp_path`/`meta_path` on every call. Cache both as struct fields.
- **M3-perf** `transcript.rs:84` — `to_string_pretty` does extra work; compact JSON is sufficient for a machine-read file.

**LOW**

- **L1-perf** `transcript.rs:74–77` — `raw.log` unbuffered writes (subsumed by H2 BufWriter fix).
- **L2-perf** `transcript.rs:95–99` — `close()` doesn't call `sync_all`; durability vs. power-loss is a deliberate trade-off, flagged for awareness.

---

### Security Reviewer

**HIGH**

- **H1-sec** `transcript.rs:21–26, 82–83` — `run_dir` accepted as unvalidated `&Path`; path traversal / symlink attack sink. **Boundary fix belongs to T4.3 caller** — not a T4.2 code change, but T4.3 must constrain `run_id` to `[A-Za-z0-9._-]` and prefix-check the canonical path before calling `create()`.
- **H2-sec** `transcript.rs:33–37, 86–90` — `.tmp` file left behind on rename failure; accumulates sensitive data on disk. **Fix**: Remove `meta.json.tmp` on any error before propagating.

**MEDIUM**

- **M3-sec** Predictable tmp name — use `O_CREAT|O_EXCL` (`create_new(true)`) to prevent symlink pre-creation.
- **M4-sec** TOCTOU between `create_dir_all` and file opens — caller (T4.3) should canonicalize + prefix-check `run_dir`.
- **M5-sec** File permissions default to umask — on Unix, set mode `0600` for files and `0700` for dir.
- **M6-sec** `raw.log` writes unbounded, unsanitized bytes including ANSI codes. No size cap.
- **M7-sec** `transcript.jsonl` has no per-run size cap; runaway child can fill disk.

**LOW**

- **L-sec-8** `Run` fields and `ToolCall.input` may contain secrets; no redaction hook.
- **L-sec-9** No `sync_all` before rename — same as perf L2.
- **L-sec-10** Serde error text in `AppError::Internal` may leak field fragments.

---

### Scope Reviewer

**HIGH** (escalation, not a code fix)

- **H1-scope** — `raw.log` flush policy (no per-write flush; flush only on `close`) was decided by the Coder. The trade-off (performance vs. raw-log completeness on crash) is an Architect decision. **Escalate to Architect.**

**MEDIUM** (escalations)

- **M1-scope** — Constructor named `create` vs `open` (monitoring spec `monitoring.md §2.5` uses `open`). Naming diverges from upstream spec. **Escalate to Architect** before T4.3 locks the API.
- **M2-scope** — `run_id` not stored on struct; monitoring spec requires it on every error/slow-flush log line. Without it T9.2 cannot add instrumentation without a breaking API change. **Adding `run_id` field to struct is included in T4.2 fix pass** to avoid T4.3 API break.
- **M3-scope** — `correlation_id` not on struct; same monitoring requirement. Deferred to T9.2 (lower urgency than `run_id`).

---

### Code Quality Reviewer

**HIGH**

- **H1-quality** `transcript.rs:29 + 82` — Atomic-write sequence duplicated verbatim between `create` and `update_meta`. **Fix**: extract `async fn write_meta_atomic(run_dir, run) -> AppResult<()>`.
- **H2-quality** `transcript.rs:32, 66, 85` — `serde_json` errors mapped to `AppError::Internal` instead of `AppError::Parse`. **Fix**: use `AppError::Parse`.

**MEDIUM**

- **M1-quality** — `close()` doc should state that callers **must** call it to flush `raw.log`.
- **M2-quality** `transcript.rs:36, 89` — `drop(tmp_file)` is unexplained; scope the block or add a comment (`// close fd before rename — required on Windows`).
- **M3-quality** — `flush` without `sync_all` in `update_meta` (same as perf M3 / sec L9).
- **M4-quality** `transcript.rs:122` — Test helper `make_run` uses `/tmp/project` (Unix-only). Use `std::env::temp_dir()` or `PathBuf::from("project")`.

**LOW**

- **L1-quality** — `FileEdit` and `Error` RunEvent variants not covered by any test.
- **L2-quality** `transcript.rs:81` — `update_meta` should take `&self` not `&mut self`.
- **L3-quality** — `to_string` + `.as_bytes()` → use `to_vec` (minor idiom).
- **L4-quality** — `expect()` messages too terse for CI diagnostics.
