# T3.2-fixes — Review fixes for useSequences/SequenceRow/SequenceList

## What was done

Applied 7 post-review fixes to `SequenceRow.tsx` and `SequenceList.tsx`. No changes
to `useSequences.ts` or test infrastructure. Updated two test assertions to match the
new empty-state text.

## How the affected components work

### SequenceRow (after fixes)

- Wrapped in `React.memo` (FIX-3) so it only re-renders when its own props change.
- Static style objects (`CONTENT_STYLE`, `NAME_STYLE`, `DESCRIPTION_STYLE`,
  `RUN_BTN_STYLE`) are now module-level constants (FIX-6), allocated once at
  module load rather than on each render. `rowStyle` stays inline because it
  derives from `selected` and `hovered` state.
- Per-property `as React.CSSProperties["fontWeight"]` casts removed; the whole
  style object satisfies `React.CSSProperties` without them (FIX-7).
- Selected state now uses `borderLeft: '3px solid var(--primary)'` with a
  matching `3px solid transparent` for the unselected case (keeps layout stable),
  replacing the previous full `border: '1px solid var(--primary)'` (FIX-2).

### SequenceList (after fixes)

- `handleRun` and `handleSelect` are now `useCallback` with stable dep arrays
  (FIX-4). Combined with `React.memo` on `SequenceRow`, this prevents unnecessary
  row re-renders when the list re-renders.
- A `useEffect` watching `projectId` resets `selectedId` to `null` whenever the
  project changes (FIX-5). This prevents a stale selection from appearing when the
  user navigates to a different project.
- Empty state text updated to include directory hint (FIX-1):
  `"No sequences found. Add .md files to .claude/sequences/ in your project."`

## Decisions made within Task scope

- For the unselected border in `rowStyle`, `borderLeft: '3px solid transparent'`
  was chosen (instead of omitting `borderLeft` entirely) to keep the row width
  stable and avoid layout shift when selection state changes. This is consistent
  with how `ProjectCard.tsx` handles similar transitions.
- `handleRun`'s `useCallback` dep array is `[]` (empty) because the function only
  calls `console.log` — no closures over state or props.
- `handleSelect`'s `useCallback` dep array is also `[]` because `setSelectedId`
  from `useState` is stable across renders (React guarantee).
