## How the affected component works

### tokens.css

All CSS custom properties live on `:root`. Components reference them as `var(--token-name)`. No JavaScript reads these values; they are purely a CSS layer. The typography scale maps spec roles directly: `--font-size-xs` (11px) for metadata, `--font-size-sm` (13px) for body/labels, `--font-size-base` (14px) for section headings, `--font-size-code` (12px) for monospace, `--font-size-lg` (18px) for screen titles/app name.

### globals.css reduced-motion

The `@media (prefers-reduced-motion: reduce)` block works in two ways:

1. CSS variable override on `:root` — any component using `transition: ... var(--duration-fast)` etc. automatically gets 0ms with no extra code.
2. Hard override on `*` with `!important` — catches any hard-coded duration that doesn't use a variable.

### App.tsx routing

`HashRouter` makes all navigation URL-hash-based (`#/`, `#/setup`, etc.), which is required in Tauri's webview environment because there is no web server to handle real path-based routing. `React.lazy` code-splits each route; the `Suspense` fallback is shown during the async chunk load. The catch-all `<Route path="*">` ensures any unknown hash redirects to Dashboard rather than a blank screen.

### Route stubs

Each stub is a single function component returning a styled `<div>` with the screen ID. This is enough to confirm routing works visually during development. All stubs will be replaced by real screen implementations in subsequent epics.

### App.test.tsx

Tests set `window.location.hash` before rendering `<App />`. `HashRouter` reads `window.location.hash` at mount time to determine the initial route. `screen.findByText` is used (async) because routes are lazy-loaded via `React.lazy`; the `Suspense` fallback renders first, then the lazy chunk resolves.

## Decisions made within task scope

- **Spacing scale trimmed to spec values only**: The prior `tokens.css` had extra values (`--space-5`, `--space-7`, `--space-9`, `--space-10`, `--space-11`) not listed in spec §1.3. These were removed to stay strictly within the defined scale. Components that need intermediate spacing can use inline px values or CSS `calc()` from the defined variables.
- **`--font-size-md: 15px` removed**: This token was in the prior file but has no matching spec role. Removed to avoid future ambiguity. All five spec roles now map to exactly one token each.
- **Dual reduced-motion strategy**: The `:root` variable override + `*` hard override is belt-and-suspenders. Future components should use the CSS variables (so they get the override for free), but the `*` rule protects against third-party components or any hard-coded value.
- **Test uses `window.location.hash`**: Directly setting `window.location.hash` before rendering `<App />` is the simplest way to test `HashRouter` routes in jsdom without restructuring `App.tsx` to accept a test-only router prop. This approach does not require `MemoryRouter` wrapping, which would change the component's production behavior.
